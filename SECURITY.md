# 🚨 Reporting a Security Vulnerability

Please **do not report security vulnerabilities through public GitHub Issues**.

Instead, report them privately using one of the following methods.

## Option 1 — GitHub Security Advisory (Preferred)

Use GitHub's private vulnerability reporting feature:

**Security → Advisories → Report a vulnerability**

---

## Option 2 — Contact the Repository Owner

If GitHub Security Advisories are unavailable, you may contact the repository owner directly through GitHub.

---

# 📋 What to Include

To help investigate the issue quickly, please include:

- Description of the vulnerability
- Steps to reproduce
- Affected project(s)
- Expected behavior
- Actual behavior
- Potential impact
- Suggested mitigation (optional)

Screenshots, logs, or proof-of-concept examples are appreciated whenever applicable.

# 🛡 Security Best Practices

When using any AI Agent in this repository:

- Never hardcode API keys or credentials.
- Store secrets in a local `.env` file.
- Never commit `.env` files to GitHub.
- Validate all external user inputs before processing.
- Review API permissions and use the least-privilege principle.
- Keep dependencies updated to their latest stable versions.
- Carefully review generated code before executing it.
- Exercise caution when enabling tools with file system, shell, or network access.
